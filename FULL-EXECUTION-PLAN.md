## 一、核心规则（强制写入Agent记忆，永久生效）

【项目固定核心规则】
1.  项目全称：领锁智能lock.club品牌日记站（sanwan.ai风格复刻+原创产品图整合+中英双语+SEO精准获客）
2.  执行主体：已创建的6个Agent（领锁司令、领锁参谋、领锁优化、领锁笔杆、领锁码农、领锁合规）
3.  执行模式：全任务分模块并发推进，每个子任务独立提交审批，无依赖的任务可同时并行执行
4.  自动审批规则：任何任务提交到项目群后，10分钟内未收到用户明确的驳回指令，默认「审批通过」，对应Agent自动进入下一环节，无需等待手动确认
5.  进度通报机制：每10分钟，所有子Agent必须在项目群内通报自身任务进度、完成情况、待审批内容、下一步计划；领锁司令负责汇总总进度，同步群内
6.  定时巡检规则：领锁司令每10分钟执行1次全任务清单巡检，核心检查：是否有任务停滞、是否有Agent闲置、是否有逾期未推进的环节，发现问题立即调度补全任务，确保项目上线前全程无中断、无闲置
7.  项目终止节点：lock.club域名正式上线、可正常访问、全功能验证通过后，进入日常迭代阶段，日常迭代仍遵循本规则

## 三、10分钟定时Corn任务配置

### 1. Corn表达式（每10分钟执行一次）
```
*/10 * * * *
```

### 2. 定时任务固定执行内容（写入任务配置）
```
1.  执行全任务清单巡检，检查所有Agent的任务进度、完成情况、是否有停滞/闲置
2.  汇总所有Agent的进度，生成《项目总进度通报》，同步到项目飞书群
3.  检查所有待审批任务，确认是否超过10分钟审批时效，超时未驳回的自动标记「审批通过」，通知对应Agent推进下一环节
4.  若发现有Agent提前完成任务、无待执行项，立即补充分配新任务，确保无闲置
5.  若发现任务阻塞、异常，立即协调对应Agent解决，同步用户
```

## 四、自动审批规则执行细则（写入所有Agent记忆）

1. 所有Agent完成任务后，必须将交付物同步到项目群，标注「【待审批】任务ID+任务名称」，同时同步给领锁司令
2. 从消息发送到项目群的时间开始计算，10分钟倒计时启动
3. 10分钟内，若收到用户明确的「驳回/修改」指令，立即按要求修改，修改后重新提交审批，重新倒计时
4. 10分钟内，若未收到用户任何明确的驳回指令，默认「审批通过」，由领锁司令通知对应Agent，自动进入下一环节
5. 涉及网站上线、域名绑定、代码终版的核心节点，无论是否超时，均需用户手动审批确认后，方可执行
