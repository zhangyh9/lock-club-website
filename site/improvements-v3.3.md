# 物联后台 v3.3 迭代改进文档
> 改进版本：v3.3 | 日期：2026-03-28

## 改进1：设备登记注册弹窗（新增设备 → 填写信息 → 绑定房间）
**文件位置**：修改 `page-device` 工具栏 + `improvements-v3.3.js`
**理由**：原系统截图02硬件物联有设备管理但缺少新增设备入口，Demo也没有设备注册功能。本次新增完整设备登记表单（类型选择→型号联动→UUID输入/扫描/自动生成→楼栋楼层绑定→房间绑定→备注），形成设备增删改查完整闭环。

## 改进2：能耗规则预览确认弹窗（saveEnergyRules 显示变更预览再确认）
**文件位置**：修改 `openEnergyRulesConfigModal` 的保存按钮 + `improvements-v3.3.js`
**理由**：原系统截图04节能风控有规则配置但saveEnergyRules直接保存没有预览。本次新增预览弹窗，显示变更配置项的原值→新值对比，确认后才执行保存，防止误配置。

## 改进3：交接班完整流程弹窗（交班人→接班人→数据核对→签字确认→打印）
**文件位置**：新增 `openFullShiftHandoverModal` 函数 + 交接班报表页面入口 + `improvements-v3.3.js`
**理由**：现有交接班只有简单的确认弹窗，缺少数据核对和签字确认的完整流程。本次新增三步骤向导（交接人确认→数据核对→签字确认），包含营收统计、房态变化、待办事项移交、双方签字、打印功能。

## 改进4：备份恢复二次确认详细弹窗（显示备份内容+风险提示+勾选确认）
**文件位置**：修改 `restoreBackup` 调用 + `improvements-v3.3.js`
**理由**：restoreBackup只有简单confirm，缺乏详细的恢复确认。本次新增独立弹窗，显示备份详情（时间/类型/大小/内容）、风险等级评估、可恢复内容清单、注意事项，勾选确认框后才能执行恢复。

## 改进5：房间维护状态设置弹窗（含维护原因/预计恢复时间/通知客人选项）
**文件位置**：新增 `openRoomMaintenanceSetModal` 函数 + 楼层页面工具栏入口 + `improvements-v3.3.js`
**理由**：现有批量维护只是简单设置状态，缺少维护原因和预计恢复时间的管理逻辑。本次新增完整表单（维护原因下拉→预计恢复时间→指派工程师→通知客人选项→自动屏蔽预订→备注），形成维护状态管理闭环。
