## 二、自动迭代机制

### 2.1 迭代Agent配置

| 任务 | 频率 | 执行内容 |
|------|------|---------|
| 内容丰富度自检 | 每日 | 检查各section内容充实度 |
| SEO关键词扫描 | 每周 | 检测关键词覆盖情况 |
| 死链/图片404检查 | 每日 | 自动修复或上报 |
| sitemap生成 | 每次push后 | 自动更新 |
| 图片同步 | 每2小时 | 自动同步Wiki图片到七牛云 |

### 2.2 迭代脚本
- `content-enricher.sh` — 内容丰富度评分（1-10分），评分低则触发通知
- `image-sync-manager.py` — 图片自动同步（见TOOLS.md）
- `keyword-research.sh` — 每月关键词更新
- `performance-check.sh` — 性能评分（Google Lighthouse）

### 2.3 手动触发点（老板决策）
- Hero轮播素材到位 → 替换占位图
- 新客户案例 → 新增到 `customers` JS数据
- 新产品上线 → 更新products页面
- SEO文章 → 增加journal页面内容

## 三、迭代执行标准

### 内容评分维度（每页1-10分）
| 维度 | 说明 |
|------|------|
| 文字充实度 | 字数/图片数量 |
| 数据可信度 | 数字前后一致 |
| CTA清晰度 | 是否有多处行动按钮 |
| 信任背书 | 徽章/认证/案例 |
| SEO优化 | title/meta/关键词密度 |
| 移动适配 | 手机端显示效果 |

### 自动评分阈值
- 评分 ≥ 8：无需改动
- 评分 5-7：自动优化（填充内容/修复链接）
- 评分 < 5：**通知老板人工介入**
