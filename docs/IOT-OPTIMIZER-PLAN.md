# 领锁物联后台精简优化执行计划
> 版本：v1.0 | 创建：2026-03-30 | 定位：强物联删PMS
> 执行频率：每10分钟一轮，每轮 ≥3 个优化点

---

## 一、战略定位原则

**一句话**：领锁物联后台 = 物联网设备管理专家，不是 PMS 传统前台系统。

### 功能优先级
| 优先级 | 定位 | 内容 |
|--------|------|------|
| 🥇 核心主业 | 物联设备监控 + 数据价值 | 设备状态监控、批量控制、故障诊断、能耗管理 |
| 🥈 核心支撑 | 报表与数据价值 | 开锁记录、告警分析、账单异常检测、运营数据可视化 |
| 🥉 辅助功能 | 传统 PMS 功能 | 入住/退房、客房清洁、员工排班、会员管理等 |

### 删除原则
- PMS业务代码（一键删除）：入住办理、退房办理、房间详情里的入住记录、清洁管理、发票管理（财务）、押金管理、夜审报表、交接班报表、会员管理、积分商城、黑名单
- 重复功能（一键删除）：侧边栏重复菜单、全局搜索空壳（删除UI或实现）
- 问题功能（删除或重做）：键盘快捷键弹窗（只展示不绑定）、批量设备控制报错（补函数或删按钮）

## 三、执行规则

### 每轮任务选取规则
1. 优先选取未完成的最小任务（10-30分钟）
2. 每轮必须包含 ≥3 个优化点
3. 功能性改进 ≥2 个（不能全是样式调整）
4. 每个优化必须有业务逻辑依据

### 任务状态追踪
- 文件：scripts/iot-optimizer/task-status.json
- 格式：每个任务含 {id, name, status: pending/doing/done, completedAt, notes}

### 优化报告格式
