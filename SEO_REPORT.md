### ⚠️ 3.4 结构化数据 (Schema.org JSON-LD)

首页包含 **大量高质量结构化数据**（已确认为优秀实践）：

| Schema类型 | 数量 | 用途 |
|-----------|------|------|
| BreadcrumbList | 2份 | 导航路径 SEO |
| Organization | 1份 | 品牌实体 |
| Product (LS02) | 4份（重复⚠️） | Google产品富片段 |
| SoftwareApplication | 3份 | SaaS平台富片段 |
| FAQPage | 1份 | FAQ富片段（People Also Ask） |
| QAPage | 1份 | 独立问答富片段（与FAQPage互补⚠️） |
| WebSite + SearchAction | 1份 | 品牌搜索框快捷入口 |
| LocalBusiness | 2份（重复⚠️） | 本地商家信息 |
| Hotel | 1份 | 酒店合作商信息 |
| Service | 1份 | 安装售后服务 |
| VideoObject | 1份 | 视频富媒体 |
| HowTo | 4份（重复⚠️） | 升级流程指南 |
| ItemList | 1份 | 核心卖点列表 |

**发现问题：**
1. **Product 结构化数据重复4次** — 同一产品出现4个不同描述的 Product schema，可能导致 Google 困惑
2. **HowTo 结构化数据重复4次** — 同一安装流程出现4个版本
3. **LocalBusiness 重复2次** — 同一商家出现2个版本
4. **FAQPage 和 QAPage 同时存在** — Google 建议同一页面只使用一种 Q&A 类型

**建议：** 合并重复的 Schema，减少冗余，保留最完整的一个版本即可。

### 🔧 优化建议（按优先级排序）

#### 🔴 高优先级

1. **补充 Twitter Card 标签**（cases/journal/tech/contact）
   ```html
   <meta name="twitter:card" content="summary_large_image">
   <meta name="twitter:title" content="页面标题">
   <meta name="twitter:description" content="页面描述">
   <meta name="twitter:image" content="https://lockclub.wangjile.cn/领锁-三大核心价值长图.png">
   ```

2. **合并重复的 Schema.org 结构化数据**
   - Product：4个重复 → 保留1个最完整的（包含 aggregateRating + reviews + offers）
   - HowTo：4个重复 → 保留1个完整版
   - LocalBusiness：2个重复 → 保留1个
   - FAQPage 和 QAPage 同时存在 → 建议只保留 FAQPage（Google 官方推荐）

#### 🟡 中优先级

3. **sitemap-images.xml 的 lastmod 未标注** — 主 sitemap 有日期，图片 sitemap 的 URL 无 lastmod，建议补充以提升图片收录优先级

4. **privacy.html 英文版缺失** — robots.txt 注释中注明 "EN privacy.html 不存在"，如需覆盖国际用户建议补充

#### 🟢 低优先级（锦上添花）

5. **keywords meta 标签已过时** — 现代 SEO 中 `<meta name="keywords">` 对 Google 排名影响几乎为零，可考虑移除或简化为3-5个核心词

6. **可考虑添加 `lang="zh-CN"` 在 html 标签上**（当前已有 `<html lang="zh-CN">`，状态 ✅）
