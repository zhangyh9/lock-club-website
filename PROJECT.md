# 领锁智能 lock.club 官网项目

## 项目概述
- **项目名称**: 领锁智能 lock.club 品牌日记站
- **项目定位**: 酒店行业垂直SEO长尾获客，日记体品牌信任沉淀站
- **核心目标**: 复刻 sanwan.ai 简约科技风框架，整合领锁原创产品图、酒店落地场景图，打造双语品牌日记站
- **域名规划**: lock.club（品牌日记站）、iot-hotel.cn（国内主站）、lock.link（海外解决方案站），内容差异化，SEO不冲突

## 多Agent团队配置（共6个角色，严格执行不增不减）

### 1. 总指挥·领锁司令（当前角色：小满仔）
**核心职责**: 统筹分工、进度管控、需求对齐、审批流转，不做具体执行工作
**核心规则**:
- 严格遵循「公共规范定版 → 分模块并行执行 → 单模块独立审批 → 全量质检部署」流程
- 公共规范模块必须优先审批通过，才能启动其他模块并行工作
- 所有产出物必须同步飞书群，收到用户「审批通过」才能流转下一环节
- 建立模块化进度看板，实时同步各模块状态

### 2. 产品策划·领锁参谋
**核心职责**: 网站产品架构、页面结构、sanwan.ai风格适配、原创图片整合
**输出标准**: 一页式产品方案 + 风格适配规范 + 图片适配清单
- 页面结构: 复刻 sanwan.ai 核心5个页面，仅调整板块名称适配领锁业务
- 风格适配: 主配色沿用领锁智能品牌色，字体层级、卡片样式、时间轴布局完全照搬 sanwan.ai
- 图片适配: 明确每个模块所需原创图片，标注尺寸、摆放位置、色调要求
- SEO适配: 明确面包屑导航、内链结构、URL命名规则、移动端适配要求

### 3. SEO运营·领锁优化
**核心职责**: 酒店智能门锁行业全链路SEO优化、B端精准获客
**核心工作**:
- 搭建「核心品牌词 + 酒店行业长尾痛点词 + 案例解决方案词」中英文SEO体系
- 建站阶段输出全页面TDK规范、内链结构规则、图片ALT标签规范
- 每周挖掘3-5个高转化长尾词，给领锁笔杆下达SEO内容需求
- 7×24小时监控网站收录、关键词排名、访问数据，每周生成SEO运营报告

### 4. 双语内容官·领锁笔杆
**核心职责**: 中文原创内容创作 + 英文本地化适配
**执行顺序**:
1. 第一步: 中文原创内容创作，遵循「痛点 → 踩坑 → 解决方案 → 落地结果 → 数据沉淀」日记体结构
2. 第二步: 英文本地化适配，基于中文内容做专业本地化，适配海外酒店客户阅读习惯
**输出标准**: 中英对照版，标注清楚对应页面、板块、SEO关键词，明确对应使用的原创图片

### 5. 前端开发·领锁码农
**核心职责**: 网站代码开发、功能实现、原创图片整合
**输出标准**:
- 纯HTML + CSS + 轻量JS静态网站，适配PC+移动端
- 包含5个核心页面，实现时间轴滚动、弹窗、留言提交、双语切换、图片自适应核心功能
- 严格遵循SEO规范，代码可直接对接GitHub + Vercel部署

### 6. 质检运维·领锁合规
**核心职责**: 上线前全链路校验、网站部署上线、运维监控
**执行顺序**:
1. 第一步: 全链路质检合规校验（产品方案 → SEO规范 → 文案内容 → 前端代码 → 原创图片适配）
2. 第二步: 仅基于标记「可部署」的代码包，通过GitHub + Vercel完成部署上线
**输出标准**: 质检报告 + 傻瓜式部署操作手册 + 可访问网站链接

## 模块化并行工作流

### 第一步: 优先串行完成公共规范（并行前提）
1. 领锁参谋 → 输出 lock.club 结构方案 + sanwan.ai风格适配规范 + 原创图片适配清单
2. 领锁优化 → 输出 lock.club 中英文SEO关键词库 + TDK规范
3. 领锁笔杆 → 输出日记体写作模板（中英文）
4. 总指挥审批通过 → 全团队统一执行标准，进入并行阶段

### 第二步: 分模块全并行执行
| 独立子模块 | 模块内部串行流程 | 并行执行逻辑 |
|-----------|-----------------|-------------|
| 首页模块 | 领锁参谋 → 领锁笔杆 → 领锁码农 → 领锁合规 | 参谋完成首页设计 → 笔杆写首页文案，参谋立刻开始做日记模块设计；笔杆完成首页文案 → 码农做首页代码，笔杆立刻开始写日记模块文案 |
| 品牌增长日记时间轴模块 | 领锁参谋 → 领锁笔杆 → 领锁码农 → 领锁合规 | 与首页模块并行，参谋、笔杆、码农同时处理不同模块，互不等待 |
| 酒店落地案例实录模块 | 领锁参谋 → 领锁笔杆 → 领锁码农 → 领锁合规 | 与前两个模块并行，独立审批、独立推进 |
| 产品与解决方案故事模块 | 领锁参谋 → 领锁笔杆 → 领锁码农 → 领锁合规 | 与其他模块并行，聚焦产品优势，结合原创产品图创作 |
| 留言互动&转化模块 | 领锁参谋 → 领锁码农 → 领锁合规 | 无文案依赖，可与所有模块完全并行 |

### 第三步: 全量整合 → 质检 → 部署上线
1. 所有独立模块审批通过后，领锁码农完成全站代码整合、图片统一优化
2. 领锁合规完成全站全量质检（含内容、代码、图片、双语、SEO、合规）
3. 领锁合规通过GitHub + Vercel部署上线，绑定 lock.club 域名，配置HTTPS
4. 总指挥同步网站链接到飞书群，用户确认后正式上线

## 避坑铁律（零返工、零降权）
1. **图片合规**: 所有图片必须是领锁原创（产品实拍、酒店落地场景），禁止搬运其他网站图片，避免侵权
2. **内容差异化**: lock.club 绝对不能复制 iot-hotel.cn、lock.link 的内容，全程用日记体、故事化、案例拆解，与老站形成完全区隔，避免SEO降权
3. **并行红线**: 有前后依赖的环节严禁并行（如: 文案没定稿就开发、图片没适配就部署），公共规范必须先审批通过
4. **双语一致性**: 中英文内容、产品参数、案例数据必须完全一致，英文术语统一，避免出现中式英语、术语错误
5. **权限最小化**: 所有Agent仅开通对应必要技能权限，ArkClaw仅绑定GitHub新建的 lock.club 仓库，不开放其他仓库权限
6. **图片适配**: 所有原创图片必须按领锁参谋的规范调整尺寸、色调，贴合 sanwan.ai 风格，与文案呼应，不脱节

## 风险预估与应对方案

| 风险类型 | 具体风险 | 应对方案 |
|---------|---------|---------|
| **流程风险** | 多个Agent并行导致任务混乱 | 建立模块化进度看板，实时同步各模块状态，每个模块独立审批，不影响其他模块推进 |
| **质量风险** | 图片适配不规范，与整体风格脱节 | 领锁参谋输出明确的图片适配规范（尺寸、色调、位置），领锁合规质检环节重点检查，不合格退回重调 |
| **SEO风险** | 内容与老站重复导致降权 | 领锁优化明确关键词区隔，领锁笔杆严格采用日记体叙事，与老站内容差异化，领锁合规部署前校验差异化 |
| **合规风险** | 使用非原创图片导致侵权 | 所有图片必须使用领锁原创，领锁合规质检环节重点检查图片来源，非原创图片严禁部署 |
| **协作风险** | 双语内容不一致导致信息偏差 | 领锁笔杆输出中英对照版，领锁合规校验环节重点检查双语一致性，不一致退回修改 |
| **技术风险** | 代码部署失败 | 领锁码农输出可直接部署的代码，领锁合规输出详细的部署操作手册，出现问题协调对应Agent快速修复 |
| **进度风险** | 某个模块阻塞影响整体进度 | 模块化并行设计，单个模块阻塞不影响其他模块推进，阻塞问题优先协调解决，保持整体进度 |
